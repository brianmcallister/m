Welcome to m, the mighty magical minimal macho menu.

So here's the deal: a lightweight jQuery select element plugin. Enough features so you can drop it in your project without wreaking everything. Smart defaults, mobile & AMD support, and most importantly, easy to write CSS for. If you're a designer that writes code, this is for you.

Basic Example

This basic example uses default settings, with no CSS. Check out the HTML, it's clean and simple to make your job styling m easier.

$('select').m();

Available Settings

classPrefix String

Pass a string to set your own, or false to have nothing.

$('select').m({classPrefix: 'class-name'});
wrapperClass String

Pass a string to set a class on the wrapper element. Useful for custom themes.

$('select').m({wrapperClass: 'class-name'});
clickToOpen Boolean

Click the title to open the dropdown list. Behaves like a regular select box.

$('select').m({clickToOpen: true});
arrowString Boolean or String

Add a unicode arrow icon. Pass true, or your own unicode or even blank string.

$('select').m({arrowString: true});

Themes

I know I said the whole point of m was to write your own CSS, but I've gone ahead and created two themes for you to use anyway. It's proof of how easy m is to style. I've included the Sass + Compass (how I've orignally written them) versions, and the compiled CSS versions (although you really should be using Sass by now, tsk tsk).

Made your own m theme? Let me know and I'll add it in here.

Monkeypatched jQuery#val

The default build of m comes with a monkeypatched (what?) val() function. There's also a build of m without the monkeypatch, if you're not into that sort of thing. If you don't use the patched val(), simply trigger your own event on the select element.

Click 'em:
var $select = $('.monkeypatch-example-select'); $select.m(); $('.monkeypatch-example-buttons').on('click', 'button', function (event) { // Just use val() like you're already doing. $select.val(event.target.className.substr(-1)); });
